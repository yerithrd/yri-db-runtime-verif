We want to have a testing for the implementation of taint analysis
because it is quite complicated to have it as a modular unit within
\yridbruntimeverif.

Therefore I need to test and implement it thoroughly since having it
working would empower users of \yridbruntimeverif to specify parametric
value in their state diagram mealy machine~(\plainSDMM).

\begin{center}
\captionof{figure}{A taint analysis ''\plainSDMM'' to test 
	parametric \plainSDMM. }
\label{fig:A_RUNNING_EXAMPLE_taintanalysis_current_test_One}
%\resizebox{\columnwidth}{!}{

\end{center}

\subsection*{Test--1}

\vspace*{0.95em}

\begin{enumerate}[$1.)$]
	\itemsep 0.95em
	
	\item After implementation of taint analysis functionality,
		the current running instance of \yridbruntimeverif needs 
		to be stopped \& restarted with a new binary.
		\scriptsize
		\begin{verbatim}
			yri_DB_RUNTIME_VERIF_uninstall.sh && yri_DB_RUNTIME_VERIF_INSTALL.SH && sudo systemctl stop yri-db-runtime-verif
		\end{verbatim}
		\small
	
	\item then start a newer instance directly without calling 
		it as a system daemon.
		\begin{verbatim}
			/opt/yri-db-runtime-verif/bin/yri-db-runtime-verif
		\end{verbatim}
	
	
	\item Then start \yeritherp.
	
	
	\item Particularly we need to check for instance that any
		string input from a \texttt{"YeritLineEdit"} GUI widget
		is coined as a tainted input except if it was also 
		passed into a sanitizer method;
		
		To test this, Tainted--Analysis functionality, we created
		a \plainSDMM using \yriqvge~:~Figure~\ref{fig:A_RUNNING_EXAMPLE_taintanalysis_current_test_One}
		illustrates this tainted analysis as a \plainSDMM.
\end{enumerate}
